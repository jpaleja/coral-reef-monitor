The basic map layout chosen for the web application was wide with a smaller height due to the nature of coral existing at the equator therefore a more horizontally focused map was deemed more appropriate. The standard features of a map were included on the web application which were zoom buttons, a scale and attribution. The data in the map includes Google Satellite and Terrain tiles, towns from naturalearthdata.com, coral locations from the UNEP World Conservation Monitoring Centre (UNEP-WCMC) and coral bleaching from Reef Base, all which are able to be ticked on and off in the control box. A search function was implemented to provide an easy way for users to navigate existing data, however, with a dropdown of coral reef names and a zoom to function that takes the user to straight to the full extent of the coral reef. This was considered appropriate due to it being highly unlikely that users knowing the names and locations of many obscure coral reefs, therefore providing an easy way to locate them. Many of the coral reefs have unreported names which are shown in the list therefore demonstrate a good use example where data could be contributed by the user in the final application.
